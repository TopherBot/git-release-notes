# git-release-notes

A tiny, pure‑Python utility that creates release notes from your Git logs.

## Installation
```bash
# Option 1: copy the single script into your repo
curl -O https://raw.githubusercontent.com/yourname/git-release-notes/main/git_release_notes.py
chmod +x git_release_notes.py

# Option 2: install via pip (once published)
pip install git-release-notes
```

## Usage
```bash
./git_release_notes.py <old-tag> <new-tag> [--output NOTES.md]
```
- `<old-tag>` – previous release tag (e.g., `v1.2.0`).
- `<new-tag>` – current release tag (e.g., `v1.3.0`).
- `--output` – optional path to write the markdown; defaults to stdout.

The script:
1. Lists commits between the two tags.
2. Parses each commit using a lightweight Conventional Commits parser.
3. Groups them by type (Features, Bug Fixes, Docs, etc.).
4. Emits nicely formatted markdown.

- **Instant scaffolding** – drop the file, run, and you have release notes.
- **No dependencies** – just the Python stdlib.
- **Automation‑ready** – pipe it into CI pipelines for automated changelog generation.
